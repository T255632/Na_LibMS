About NaLib

NaLib is a comprehensive and user-friendly Library Management Information System (LMIS) designed to streamline the management of library resources, users, genres, and transactions. Built with a modern, modular architecture, NaLib provides a robust and scalable solution for libraries of all sizes.

Key Features & Technologies

  • Modular Design: Separation of Frontend, Backend, Database, and Security layers for maintainability and scalability.
  • Client/Server Architecture: Utilizes a client-server model for efficient data access and processing.
  • Frontend: HTML, CSS, and JavaScript provide an intuitive user interface. Communication with the backend is handled via HTTP requests using Fetch API.
  • Backend: A Flask application handles API requests, business logic, and database interactions. Implements JWT-based authentication and Role-Based Access Control (RBAC).
  • Database: PostgreSQL manages user accounts, lending records, membership information, and audit logs. MongoDB stores the library catalog and metadata for genres and formats.
  • Security: JWT tokens ensure secure user authentication. RBAC restricts access based on user roles. Data encryption is used both in transit (HTTPS) and at rest.
  • Database Enhancements: PostgreSQL Triggers and Stored Procedures automate tasks such as updating resource statuses, logging actions, enforcing borrowing rules, and managing membership updates.

Functionality Highlights

  • Resource Management: Comprehensive cataloging, including support for various formats (hardcopy/electronic) and genres.
  • User Management: Manage user accounts, roles (Admin, Staff, Member), and permissions.
  • Lending Management: Efficiently handle borrowing and returning transactions, track overdue items, and enforce borrowing rules.
  • Reporting: Generate reports on borrowed resources, overdue items, member statistics, and more.
  • Search & Browsing: Easily search and browse the library catalog.
  • Role-Based Access: Different user roles have access to specific functionalities based on their permissions.
